# 动态路书分享方案

目标：让家里人不用科学上网，也能稳定打开这个项目。

## 不建议继续用的方案

- `cloudflared quick tunnel`
- `localhost.run`
- `localtunnel`

原因：

- 域名临时变化，不适合发给爸妈长期保存
- 稳定性差，容易偶发打不开
- 访问链路不在你掌控里，排查成本高

## 推荐顺序

### 1. 同一 Wi-Fi / 家里局域网访问

适合：

- 你和爸妈在同一个网络环境里
- 只是临时演示或出发前在家里看

做法：

1. 本机运行 `npm run dev` 或 `npm run start`
2. 查本机局域网 IP，比如 `192.168.1.20`
3. 爸妈直接打开 `http://192.168.1.20:3000`

优点：

- 最简单
- 不需要公网域名
- 不需要任何科学上网

缺点：

- 只能在同一个局域网里用

### 2. 你自己的云服务器或家宽 DDNS

适合：

- 想给爸妈长期使用
- 需要固定地址

做法：

1. 买国内可访问的云服务器，或者用家里宽带做端口映射
2. 用 `PM2` 或系统服务长期跑 `next start`
3. 配一个固定域名
4. 用 `Nginx` 反代到项目端口

优点：

- 地址固定
- 你自己可控
- 稳定性远高于临时 tunnel

缺点：

- 需要一点部署成本
- 如果要用高德线上地图，域名还要加到高德白名单

### 3. 国内静态托管

适合：

- 主要展示路线和图片
- 地图可以接受离线底图，不强依赖高德在线交互

思路：

- 把项目进一步改成静态导出可用
- 部署到国内静态站点平台或对象存储静态网站

优点：

- 访问快
- 很省心

缺点：

- 需要继续调整 Next.js 输出方式
- 高德在线地图能力会受限制

## 这个项目当前最现实的落地建议

如果目标是“爸妈稳定打开”：

1. 先把页面部署到你自己的国内可访问地址
2. 高德地图当增强项，不当硬依赖
3. 就算高德配置失效，页面也必须还能看时间线、图片和离线地图
